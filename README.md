## 能回答哪些问题

这个 Skill 最擅长的不是“按钮怎么点”，而是判断问题到底卡在流量、素材、归因、变现、组织，还是金融风控闭环。典型问题包括：

- 为什么单价突然变了，问题更像预算、出价、流量池变化，还是后端回收变了。
- 为什么平台后台和 MMP / AppsFlyer / SAN / SKAN 数据对不上，先该查口径还是查投放动作。
- IAA、IAP、订阅、短剧、工具、金融这几类业务，投放目标和回收周期该怎么定。
- 素材还能不能继续扛，素材复用会不会带来疲劳、降权或流量池错配。
- 小红书、快手、巨量、广点通这类内容平台，怎么同时看分发、互动、成交和承接链路。
- ASA / ASO / 华为 / vivo / oppo / 小米 / 苹果应用商店，为什么有量但质量和长期回收不稳。
- Meta / Google / UAC / AAA 这类平台算法变化后，老经验为什么失效，应该怎么重建判断。
- 团队为什么总定位不到真问题，复盘、述职、招聘负责人和投手应该怎么拆标准。
- 海外现金贷 / 助贷 / 消金项目，该不该进某个国家，先看牌照、利率、支付、坏账还是市场热度。
- 海外风控冷启动该先搭哪几个最小单元，模型为什么几个月就失灵，欺诈和毛利为什么会吞掉增长。

## 应该如何使用

最好把它当成“投放诊断和决策助手”，不是“泛泛投放百科”。

比起问“广告该怎么投”，更适合这样问：

- Meta 预算一加大，CPM 和 CPA 一起涨，这更像是流量池变了还是用户变差了？先看什么？
- 巨量 IAP 短剧前端 CPA 还行，但首日 ROI 不达标，按主因优先级帮我拆。
- 小红书想让粉丝和成交一起涨，先抓哪几个点？
- vivo 应用商店关键词计划首日 ROI 不错，但留存不稳，这更像是哪层没拆开？
- 我们在招投放负责人，怎么分辨他是吃过红利，还是逻辑真的强？

提问时如果顺手补这些信息，回答会准很多：

- 平台：Meta / Google / 巨量 / 小红书 / 华为商店...
- 业务模型：IAA / IAP / 混合变现 / 短剧 / 工具 / 电商 / 金融...
- 当前症状：没量 / 有量但贵 / 前端还行但回收差 / 数据打架
- 时间范围：今天 / 最近 3 天 / 最近 7 天 / 放量后
- 关键指标：CPA / CPM / CTR / CVR / ROI / ROAS / 留存 / 付费率
- 团队分歧：现在内部有人想提价 / 复制 / 换素材 / 加预算

它默认会按这个顺序回答：

- 先说结论
- 再判断问题更像哪一层
- 再解释为什么
- 再告诉你先看什么
- 再告诉你现在最不建议乱做什么
- 最后给一个处理顺序

一句话理解：

这套 Skill 最擅长的不是教你“点哪个按钮”，而是先判断你的问题到底属于流量、素材、回收、归因、内容分发、应用商店，还是管理协同，再给你一个有优先级的处理路径。

## 使用建议

- 当用户的问题本质是投放、买量、流量、素材、归因、回收、平台算法或团队管理时，优先启用这个 Skill。
- 如果问题明确涉及海外现金贷 / 助贷 / 消金 / 金融科技出海，默认同时加载金融风控纵深层，而不是只按媒介动作回答。
- 显式唤醒时，优先使用 `$SST-TOUFANG-SKILL`。
- 回答时不要只给动作，先判断问题属于哪一层，再给优先排查项和不建议动作。
- 先分清是竞价广告平台、内容推荐平台、应用商店平台，还是金融风控垂类问题，再决定怎么拆题。
- 遇到平台后台、MMP、SAN、SKAN 数据不一致的问题，先对齐统计口径，再讨论优化动作。
- 遇到金融垂类问题，先把牌照、利率、支付、坏账、毛利和欺诈结构算清楚，再谈放量与增长。
