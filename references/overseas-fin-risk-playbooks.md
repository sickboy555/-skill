# 海外金融风控作战卡

这份文档把 `路费的海外风控笔记` 里高频出现的海外金融风控问题压成一组“先判断什么、先别做什么、先怎么动”的作战卡。

它主要服务于 `现金贷 / 助贷 / 消金 / 反欺诈 / 风控组织` 场景，和 [`problem-playbooks.md`](./problem-playbooks.md) 的通用投放作战卡配合使用。

## 1. 这个国家能不能做 / 现在值不值得进

### 更像是什么

- 真问题通常不是“市场大不大”，而是“牌照、利率、支付、身份、催收、汇率和竞争能不能让你活下来”。
- 下载榜和活跃榜只能说明关注度，不能说明利润质量。

### 先看什么

1. 牌照路径、外资进入方式、利率上限。
2. 本地身份认证、支付、征信、三方数据基础设施。
3. 节日、宗教、语言、地区差异和催收边界。
4. 竞争拥挤度、生态玩家强度、流量获取成本。
5. 资金成本、税费、利润汇回和汇率波动。

### 当前最不建议做的事

- 只看“市场热度高”就冲进去。
- 把别国经验直接复制过去。

### 如果是我来处理

1. 先排除牌照和利率逻辑不成立的国家。
2. 再看本地数据和支付基础设施够不够支撑冷启动。
3. 最后再决定是自己做、合作切入，还是先小规模试点。

## 2. 海外现金贷冷启动，风控先搭什么

### 更像是什么

- 冷启动阶段最怕不是模型不够复杂，而是闭环没跑通。
- 真正决定生死的，是最小单元能不能在小样本里成立。

### 先看什么

1. 风控画布：客群是谁、要防谁、坏账容忍度到哪。
2. 基础数据：本地身份、三方数据、设备数据、支付链路能拿到哪些。
3. 首贷规则：先用哪些规则卡住明显高风险人群。
4. 人工复核与样本回流有没有安排。
5. 通过率、坏账率、放款量的最小阈值怎么设。

### 当前最不建议做的事

- 一上来就做大而全模型。
- 把其他国家的规则包整套搬过来。

### 如果是我来处理

1. 先用风控画布把产品、客群和坏账容忍度讲透。
2. 再搭规则优先的首贷策略和反馈回路。
3. 跑通小流量后，再决定要不要上更复杂的模型和更多数据。

## 3. 量起来了，但坏账 / 回收突然恶化

### 更像是什么

- 这更像增长和风控没对齐，不只是后端“运气不好”。
- 也可能是渠道结构变了、欺诈升级了、节日周期来了，或者定价和额度没跟上。

### 先看什么

1. 最近获客渠道、人群结构、投放节奏是否明显变化。
2. 欺诈信号、设备异常、多人共机、多头借贷是否抬头。
3. 额度、期限、定价是否对新客群过于激进。
4. 贷后触达、支付成功率、催收节奏有没有变化。
5. 是某个国家、某个渠道、某类人群先恶化，还是全局恶化。

### 当前最不建议做的事

- 只把锅甩给投放团队或催收团队。
- 还没分清是哪段链路坏了，就全量收紧所有策略。

### 如果是我来处理

1. 先把渠道、人群、欺诈、贷后四层拆开。
2. 找出最早变坏的那一层和那一组样本。
3. 再决定是收紧准入、调整额度定价，还是先堵欺诈和修贷后。

## 4. 模型上线几个月就失灵

### 更像是什么

- 更像数据和人群分布漂移，不是“模型突然没脑子了”。
- 也可能是三方数据源衰减、节日季节性、欺诈手法变化，或者上线后缺少监控。

### 先看什么

1. PSI、KS、AUC、通过率、拒绝率和坏账率各自怎么变。
2. 哪个国家、哪个渠道、哪个客群先漂了。
3. 三方数据覆盖率和字段稳定性有没有变。
4. 是否碰上节日、政策、投放结构变化或黑产升级。
5. 线上规则和模型阈值有没有被临时改乱。

### 当前最不建议做的事

- 不做归因就直接全量重训。
- 把规则、模型、定价一起乱改。

### 如果是我来处理

1. 先定位是样本漂移、特征失效，还是输入数据脏了。
2. 再用挑战者模型或小样本实验验证修复方向。
3. 确认有效后再灰度放大，不要一次性全量切换。

## 5. 毛利账面很好看，但真实利润很薄

### 更像是什么

- 这通常不是“财务算错了”，而是你把坏账、汇率、税费、催收和获客的坑漏掉了。
- 也可能是高息产品在用坏账和促销偷偷补贴规模。

### 先看什么

1. 利差之外，还算没算资金成本、税费、汇率和合规成本。
2. 获客渠道有没有把坏账一并放大。
3. 客群结构是不是正在往高风险人群倾斜。
4. 贷后和催收成本有没有被低估。
5. 是哪类产品、哪个区域、哪段时间毛利最容易塌。

### 当前最不建议做的事

- 只看年化利率和放款规模。
- 把毛利问题简单理解成“再多放点就摊薄了”。

### 如果是我来处理

1. 先把毛利拆成可追踪的成本项。
2. 找出真正吃利润的第一大项和第二大项。
3. 再决定该改产品结构、渠道结构、风控阈值还是贷后策略。

## 6. 设备 / 人脸 / 身份欺诈变多了

### 更像是什么

- 黑产在升级，不是简单“多上一个验证码”就能解决。
- 很多欺诈会同时改 IMEI、改 Android ID、改机型、P 证件、做鬼脸和多账户操作。

### 先看什么

1. 哪类欺诈先抬头：改机、脸部伪装、多人共机，还是多头借贷。
2. 底层设备信息、传感器、陀螺仪、电池曲线有没有异常共性。
3. 人脸、年龄、性别和证件一致性是否经常对不上。
4. 是某个国家、某个机型、某个渠道更集中，还是全局泛滥。
5. 当前是硬拒、人工复核还是灰度提示，误杀成本多大。

### 当前最不建议做的事

- 只看表层设备参数。
- 让大模型直接做最终硬拒裁决。

### 如果是我来处理

1. 先补底层设备和行为信号。
2. 再让大模型做辅助识别和人工复核排序。
3. 误杀成本高的环节先灰度，不要一刀切上线。

## 7. 风控部门 OKR / 招聘 / 复盘怎么搭

### 更像是什么

- 问题往往不是没人干活，而是目标只盯坏账率，缺了数据、流程、定价和本地化维度。
- 招错人也常常不是简历少，而是标准太虚，只看年限和平台经历。

### 先看什么

1. OKR 是否覆盖模型、特征、三方数据、定价、流程和本地化。
2. 周进度、月复盘、季度校准机制有没有固定下来。
3. 候选人能不能讲清失败案例、复盘逻辑和市场适配判断。
4. 团队有没有把投放、产品、贷后和风控串成闭环。

### 当前最不建议做的事

- 只拿坏账率做唯一目标。
- 只看候选人的放款规模或过往头衔。

### 如果是我来处理

1. 先按模型、数据、定价、流程拆目标。
2. 再配周节奏和月复盘，保证反馈闭环。
3. 招人时优先看本地化理解、拆题能力和协同能力。
