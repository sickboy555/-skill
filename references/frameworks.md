# 投放方法论提炼

这份文档把 `何俊杰`、`潮州痞子蔡` 与新增的 `路费的海外风控笔记` 三批语料里高频重复的判断压缩成可复用框架。回答问题时按需调用，不必全量复述。

## 一、底层认知

### 1. 投放不是只调账户

- 投放效果是产品、定价、内容、素材、定向、媒体规则、回收设计、团队执行共同作用的结果。
- 只盯账户动作，很容易误诊。
- 很多“投放问题”其实是产品和商业化设计问题。

### 2. 先分渠道类型，再谈动作

- 竞价广告平台，更看预算、出价、素材、事件目标和人群结构。
- 内容推荐平台，更看定位、内容形式、互动结构、审核约束和变现闭环。
- 应用商店平台，更看关键词、分发、榜单 / ASO、人群包、商店算法和自然量联动。

### 3. 投放是在平台规则内找最优解

- 不是和平台对着干。
- 不是简单靠技巧骗量。
- 是理解平台偏好、流量池、算法排序，然后让自己的内容和目标更匹配。

### 4. 更理想的状态是“系统指导买量”

- 不是只靠投手凭感觉拍脑袋。
- 要通过后端埋点、行为统计、LTV 预估、事件分层，反过来指导计划去留、预算增减、素材方向和放量节奏。

### 5. 曝光质量、转化质量、回收质量、口径质量是四件事

- 曝光便宜，不代表用户值钱。
- 前端转化漂亮，不代表长期 ROI 一定成立。
- 某个计划能起量，不代表它适合放量或适合长期回收。
- 平台和三方数据对不上，也不等于投放本身一定出了问题。

## 二、诊断总轴

### 1. 先分清五种常见局面

- 没量：重点看平台门槛、账户基础、素材初始竞争力、预算门槛、阶段是否匹配。
- 有量但贵：重点看 CTR、CVR、素材疲劳、定向错配、页面承接、流量池变化。
- 前端还行但回收差：优先怀疑后端变现、回收周期、LTV 预估、劣质人群、错误放量。
- 数据互相打架：优先看平台口径、MMP 口径、归因窗口、时区、汇总逻辑和回传配置。
- 团队总找不到问题：优先怀疑诊断框架错了，或者数据体系与管理节奏出了问题。

### 2. 再分清是前端问题、后端问题，还是口径问题

- 前端问题常见表现：不起量、CTR 低、CVR 低、成本高、素材衰退快。
- 后端问题常见表现：前端成本看似正常，但回收差、ROI 差、长周期不达标、量级放大后质量掉。
- 口径问题常见表现：平台和三方数据差异大、当天数据极不稳定、平台异常回传、SAN / SKAN 统计不一致。

### 3. 内容平台和应用商店要加一层专属判断

- 内容平台先看定位、人群、互动和审核，再看投放与放量。
- 应用商店先看关键词、分发、自然量、素材展示位、人群包和类目差异，再看 CPA 或 ROI。

### 4. 阶段思维非常重要

- 冷启动 / 测试阶段：核心是验证方向，不是强求完美 ROI。
- 起量阶段：核心是找到可放大的素材和计划，同时看人群质量。
- 放量阶段：核心是后端数据支撑，不是简单加预算。
- 成熟阶段：核心是节奏、结构、素材轮换和回收稳定性。
- 衰退阶段：核心是识别到底是素材、计划、流量池、内容定位还是产品承接出问题。

## 三、关键专题

### 1. 预算 / 出价影响的不只是单价

- 小预算更像压着系统先找核心人群，单价通常更稳。
- 大预算会加快探索速度，更容易带来不精准曝光。
- Facebook 中，高 CPM 不一定是坏事，有时只是系统学到了更贵但更准的人群。
- Google 中，高出价常常意味着版位更广、人群更杂，不一定更值钱。
- 所以预算 / 出价类问题，不能只盯“贵不贵”，要看拿到的到底是什么人。

### 2. 素材是最大的定向，素材复用也可能是最大的坑

- 素材决定吸引谁、进什么流量池、拿到什么样的前端和后端质量。
- 素材不能只看 CTR，还要看 CVR、留存、事件质量、回收、生命周期。
- 老素材复用、跨账户复用、跨平台复用，都要重新评估。
- 同一素材跨产品、跨端、跨平台硬复用，可能引发频控、疲劳、用户反感和系统降权。
- 有条件的团队应该做素材系统：
  - 标准命名
  - 素材库
  - 标签体系
  - 国家 / 语言 / 渠道 / 目标 / 受众维度分析
  - 素材生命周期和衍生关系追踪

### 3. 归因 / SAN / SKAN / MMP

- 平台和 MMP 数据对不上，本来就是常态，不是先默认谁“偷量”。
- 常见差异来源：
  - 汇总时间不同
  - 时区不同
  - 归因窗口不同
  - click / view 规则不同
  - SAN 与普通网络的回传机制不同
- SAN 的重点是：平台自己掌握一部分归因判断权，和普通网络先上报点击再由 MMP 匹配不同。
- SKAN 的重点是：隐私优先，数据延迟、粒度下降、postback 限制、隐私门槛和 conversion value 设计都会影响判断。

### 4. 内容平台不是“换了个名字的效果广告”

- 小红书、快手这类平台，定位和内容结构往往比短期放量动作更前置。
- 评论、互动深度、种草氛围、内容重复识别，会直接影响分发质量。
- 涨粉、转私域、卖货、带咨询，不是一个目标，别混着用同一套动作。
- 低粉不一定差，关键看高质量人群和变现闭环。

### 5. 应用商店投放是一个独立物种

- 应用商店不只是搜索广告，也不只是买下载。
- 关键词、普通分发、人群包、榜单、ASO、自然量、评论和展示素材，经常一起作用。
- 苹果、华为、vivo、oppo、小米的机制和人群质量差异很大，不能拿一个平台经验硬套另一个。
- 很多时候单一计划无法同时兼顾首日 ROI、付费率、注册成本和留存，需要多计划对冲。

### 6. 混合变现 IAA + IAP 不能只用一个脑子看

- 常见模式有三种：
  - 纯投 IAP，IAA 当送的
  - 纯投 IAA，IAP 当补充的
  - 两边都重要，需要混投或拆投的
- 混合变现的关键不是“都要”，而是先判断哪一部分是主引擎。
- 还要进一步拆人群：
  - 付费人群
  - 广告人群
  - 潜在付费人群
- IAA 回收通常更慢，如果全按 D0 ROAS 看，容易把投放目标逼歪。

### 7. 管理、招聘和组织决定了投放上限

- 团队定位不到问题，通常不是只因为一线不努力，而是：
  - 管理层不俯身看数
  - 数据体系不完整
  - 激励和考核把动作逼变形
  - 投手没有试错空间
  - 产品、运营、设计和投放没有闭环
- 招投手和负责人时，逻辑拆解能力、复盘能力、数据归因能力，经常比“做了几年、花过多少钱”更值钱。
- 一个像样的述职 / 复盘，不该只写消耗和 revenue，还要写：
  - 对比同行
  - 自己的进步
  - 行业和平台趋势
  - 下一阶段的具体计划

### 8. 平台算法和 AI 正在改写老经验

- 平台越来越看语义、行为、上下文信号，不再完全依赖人工标签。
- 这会直接削弱两类旧优势：
  - 极度依赖精准定向的打法
  - 靠少量老素材吃很久的打法
- 所以遇到“以前能跑、现在跑不动”的问题，要先考虑算法范式变了，不要只怪投手动作不到位。

### 9. 金融风控垂类要额外补一层市场、毛利和反欺诈判断

- 问题一旦涉及现金贷、助贷、消金、牌照、利率、支付、坏账、贷后，就不能只按投放问题回答。
- 真正的生死线经常不是起量，而是 `市场准入 + 毛利结构 + 欺诈结构 + 贷后承接` 能不能同时成立。
- 冷启动阶段，先跑通风控画布、基础数据和首贷策略三个最小单元，不要一上来追求大而全模型。
- 毛利要拆资金成本、坏账、获客、支付、催收、汇率和税费，不要把高利率误当成高利润。
- 设备改机、鬼脸、人证不一致、多头借贷这些问题，本质上都是“欺诈结构变了”，不是单个模型分数就能兜住。
- 遇到这类题，优先补看 [`overseas-fin-risk-frameworks.md`](./overseas-fin-risk-frameworks.md)。

## 四、常见误判

- 低 CPA 不等于高 LTV。
- 高 CPM 不等于坏人群。
- 平台和三方数据不一致，不等于平台偷量。
- 起量素材，不等于长寿素材。
- 内容平台粉丝多，不等于变现效率高。
- 应用商店首日 ROI 好，不等于长期质量稳。
- 高利率，不等于高毛利。
- 市场热门，不等于现在值得做。
- 放款量上去了，不等于坏账和贷后还能兜住。
- 成本波动，不等于立刻提价。
- 团队业绩差，不等于先怪投手。

## 五、回答时最值得高频提醒的点

- 先别急着提价。
- 先别把锅全甩给素材。
- 先看是不是口径没对齐。
- 先分清前端和后端。
- 先分清渠道类型，不要把不同平台当同一套打法。
- 涉及金融垂类，先把牌照、利率、支付、毛利、坏账和欺诈结构算清楚。
- 先看当前是不是做了阶段不匹配的动作。
