# 海外金融风控纵深框架

这份文档把 `路费的海外风控笔记` 135 篇新增语料压缩成一套适用于 `现金贷 / 助贷 / 消金 / 高风险出海获客` 的垂类框架。

它不是替代 [`frameworks.md`](./frameworks.md) 的通用投放框架，而是在问题涉及金融业务、牌照、利率、支付、额度、坏账、贷后、反欺诈、模型迭代时，要额外叠加的一层“金融风控视角”。

## 一、什么时候要切换到这层视角

- 用户在问 `现金贷 / 助贷 / 消金 / 金融科技出海 / BNPL`。
- 用户在问 `牌照 / 利率上限 / 支付通道 / 征信覆盖 / 三方数据 / 本地监管 / 催收边界 / 资金成本 / 汇率`。
- 用户在问 `贷前准入 / 额度 / 定价 / 复贷 / 贷后 / 逾期 / 反欺诈 / 设备指纹 / 活体 / 鬼脸 / 多头借贷`。
- 用户在问 `风控冷启动 / 模型上线后失灵 / A/B 测试 / 风控 OKR / 风控团队搭建 / 风控年终复盘 / 风控 AI 应用`。
- 用户表面在问投放或增长，但本质已经落到 `坏账吞噬利润`、`高风险客群质量下滑`、`某个国家到底值不值得做`。

## 二、先补一个总判断

- 这类业务的真问题通常不是“能不能起量”，而是“起量之后能不能活下来”。
- 现金贷 / 助贷不是把投放和风控拆开的两套系统，而是一条从获客到放款、从定价到贷后、从反欺诈到利润的闭环。
- 所以这类问题不能只给媒介动作，至少要同时看 `市场`、`风控`、`毛利` 和 `组织` 四层。

## 三、七层判断轴

### 1. 市场与监管适配层

- 先看这个国家能不能做，再看这个季度值不值得进。
- 重点不是市场热不热，而是 `牌照路径`、`利率上限`、`外资进入方式`、`支付基础设施`、`身份认证基础设施`、`催收边界` 是否支撑业务闭环。
- 宗教节日、语言、地方方言、节日消费与还款习惯，本身就是风险变量，不是“本地化装饰”。
- 对金融业务来说，下载榜和活跃榜只是表象；真正要看的是 `竞争拥挤度`、`生态玩家强度`、`资金环境`、`税费和利润汇回成本`。

### 2. 商业模型与毛利层

- 高利率不等于高毛利，真实利润要拆成 `资金成本 + 风险成本 + 获客成本 + 支付成本 + 催收成本 + 汇率成本 + 税费 + 合规成本`。
- 毛利不能只按单笔利差算，要按 `产品结构`、`渠道结构`、`区域结构`、`客户结构`、`时间节奏` 来拆。
- 很多看起来“利差很肥”的市场，真正被吃掉利润的往往是坏账、促销、汇率、税费和本地化运营成本。
- 先分清产品里的“赚钱主力”和“拉新工具”，否则高通过率和高放款量可能只是在加速亏损。

### 3. 冷启动最小单元层

- 冷启动阶段不要追求“大而全”，先跑通最小闭环。
- 第一个最小单元是 `风控画布`：先想清楚防谁、赚谁的钱、坏账容忍度到哪、老板要速度还是稳收益。
- 第二个最小单元是 `基础数据清单`：本地身份体系、三方数据、设备数据、GPS、通讯录、征信、支付和行为数据，哪些真能拿到，哪些只是 PPT。
- 第三个最小单元是 `首贷策略包`：规则、人工复核、样本回流、通过率和坏账阈值先跑通，再谈复杂模型。
- 冷启动不是把别国规则搬过来就算完成，本地数据基础设施和欺诈结构一变，老经验就会反噬。

### 4. 反欺诈与设备身份层

- 黑产不是抽象概念，要拆成 `证件造假`、`脸部伪装`、`改机`、`设备农场`、`多头借贷`、`团伙撞库`。
- 设备指纹不要只看可篡改的表层参数，优先看底层序列号、传感器噪声、电池曲线、陀螺仪和系统级硬件信息。
- 人脸、年龄、性别和证件一致性，很多时候更适合做 `前置提示 + 人工复核`，而不是一刀切硬拒。
- 设备、活体、行为、三方数据要组合看，单点指标容易被黑产迅速学习和绕过。

### 5. 模型与策略迭代层

- 模型上线不是结束，很多问题死在“上线之后没人持续看”。
- 先看分群漂移、数据源覆盖变化、节日季节性、外部供应商波动、欺诈手法升级，再决定要不要重训。
- 不要迷信一次性大模型，冷启动阶段很多时候是 `规则 + 简单模型 + 快速实验` 更稳。
- 面对不确定市场，实验文化比“专家拍板”更值钱。小样本灰度、72 小时淘汰、挑战者模型，往往比全量重构风险更小。

### 6. 增长与风控耦合层

- 金融类增长不是把量买进来就结束，量的结构会直接决定坏账结构、复贷结构和催收压力。
- 如果是生态型玩家，要额外看 `场景闭环` 和 `自有数据护城河`；如果是纯买量玩家，要对获客成本和坏账质量更敏感。
- 很多“量有了但利润没了”的问题，本质不是投放失误，而是风控、定价、产品要素和获客渠道没有一起调。
- 一个市场是否值得抢，不只看下载和月活，还要看有没有稳定的高质量获客入口、是不是只能靠高价流量硬推。

### 7. 组织、OKR 与 AI 层

- 风控部门的 OKR 不能只盯坏账率，要覆盖 `模型优化`、`特征衍生`、`三方数据管理`、`定价策略`、`流程效率`、`本地化适配`。
- 真正有效的管理节奏通常是：周进度、月复盘、季度策略校准，而不是年底一次性追责。
- 风控岗位招聘，优先看 `问题拆解`、`失败复盘`、`本地市场理解`、`跨部门协同`，不是只看做过几年和花过多少钱。
- AI 在这类场景里更适合做 `短信打标`、`翻译`、`鬼脸检测辅助`、`年龄/性别一致性提示`、`报表和复盘初稿`，不要直接把它当最终裁决器。

## 四、这批语料带来的高频提醒

- 先看国家，不要先套中国经验。
- 先算毛利，不要先幻想高利率。
- 先跑最小闭环，不要先堆大而全模型。
- 先看欺诈和坏账结构，不要把问题都算成投放质量。
- 先做灰度实验，不要全量拍脑袋。
- 先把组织节奏和反馈闭环搭起来，不要只在结果差时追责。
