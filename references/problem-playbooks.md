# 高频问题作战卡

这份文档把 `何俊杰`、`潮州痞子蔡` 与新增的 `路费的海外风控笔记` 三批语料里反复出现的高频问题压成一组“先判断什么、先别做什么、先怎么动”的作战卡。

如果问题明确涉及海外现金贷 / 助贷 / 消金 / 反欺诈，请和 [`overseas-fin-risk-playbooks.md`](./overseas-fin-risk-playbooks.md) 搭配使用。

## 1. 预算 / 出价 / 单价突然变了

### 更像是什么

- 可能是曝光池变了。
- 可能是学习节奏被你推快了。
- 也可能只是系统开始吃更贵但更准的人群了。

### 先看什么

1. 当前是不是测试期 / 冷启动期。
2. 最近是否明显加过预算、改过出价、换过目标、换过素材。
3. CPM、CTR、CVR、后端回收分别怎么变。
4. 版位和人群结构有没有变化。

### 当前最不建议做的事

- 看见单价变贵就立刻提价。
- 还没分清曝光质量和转化质量，就给计划判死刑。

### 如果是我来处理

1. 先分清是“曝光更散了”还是“转化用户更差了”。
2. 再看当前预算是否把系统推到了不适合的探索强度。
3. 最后决定是收预算、稳几天、拆计划，还是补新素材。

## 2. 平台后台和 MMP / 三方数据对不上

### 更像是什么

- 多数时候不是谁在偷量，而是口径没对齐。
- 也可能是 SAN / SKAN 机制差异、回传配置问题、时区问题。

### 先看什么

1. 平台和三方看的时间段是不是一样。
2. 时区是不是一样。
3. 归因窗口是 1 天、7 天，还是 click / view 混算。
4. 是 SAN 还是普通网络。
5. 回传事件、事件映射、deep link / referrer 配置有没有问题。

### 当前最不建议做的事

- 只看半天数据就说平台崩了。
- 没对齐口径就去大改投放动作。

### 如果是我来处理

1. 先对齐时间、窗口、时区。
2. 再分清平台口径和三方口径分别在统计什么。
3. 最后再判断是统计差、归因差，还是技术回传故障。

## 3. IAA / IAP / 混合变现该怎么投

### 更像是什么

- 先别想着“全都要”，先判断产品主引擎在哪。
- 真问题常常不在投放动作，而在回收周期、商业化设计和人群拆分。

### 先看什么

1. IAA 和 IAP 各自贡献占比。
2. 回收周期是 D0、D3、D7 还是更长。
3. 当前回传的是付费、广告、还是混合 ROAS。
4. 产品内有没有“潜在付费用户”的中间行为。

### 当前最不建议做的事

- 用一个目标硬吃两套变现逻辑。
- 只按 D0 ROAS 判断所有模式。

### 如果是我来处理

1. 先分产品更偏 IAA、IAP，还是双强。
2. 再分人群：付费人群、广告人群、潜在付费人群。
3. 最后决定是混投、拆投，还是混合回传但拆考核。

## 4. 素材还能不能继续扛 / 素材复用有没有副作用

### 更像是什么

- 素材问题不只是“好不好看”，而是现在筛进来的还是不是你想要的人。
- 老素材跑不动，未必是投手懒，也可能是流量池和平台算法阶段变了。
- 双端、跨产品、跨平台硬复用同一素材，可能会直接触发频控、疲劳和系统降权。

### 先看什么

1. 素材当前承担的是起量任务、筛选任务，还是回收任务。
2. CTR、CVR、留存、后端回收、生命周期分别怎么变。
3. 不同国家、语言、平台、受众下是不是同样疲劳。
4. 老素材是否被跨账户、跨平台、跨目标复用太久。

### 当前最不建议做的事

- 因为它以前好用，就一直硬扛。
- 只看 CTR 说素材还行。
- 同一套素材横着复制到所有端和所有计划里。

### 如果是我来处理

1. 先看这个素材筛进来的人还值不值钱。
2. 再判断它是该淘汰、做衍生，还是换个使用场景。
3. 如果存在复用冲突，优先把不同端、不同目标、不同人群的素材拆开。

## 5. 小红书 / 内容平台流量和成交想一起涨

### 更像是什么

- 真问题往往不是“内容发少了”，而是定位、人群和变现闭环从一开始就没对齐。
- 这类平台不是纯竞价流量池，互动质量和平台氛围本身就影响分发。

### 先看什么

1. 账号 / 内容定位是不是清楚。
2. 想要的是涨粉、私域、成交，还是广告变现。
3. 内容的评论、收藏、点赞结构分别怎样。
4. 有没有触碰重复、抄袭、敏感词、跨平台导流等审核风险。

### 当前最不建议做的事

- 上来就把“涨粉”当唯一目标。
- 直接照抄别人爆文或别的平台内容。
- 还没看清内容承接，就只想着投流放大。

### 如果是我来处理

1. 先把定位和变现方式捆在一起看。
2. 再看互动信号，尤其是评论质量。
3. 最后再决定是补内容形态、补投流，还是先改承接链路。

## 6. 应用商店投放有量，但质量和回收不稳定

### 更像是什么

- 多数时候不是单一平台“量不行”，而是计划类型、关键词 / 分发结构、商店算法和后端目标没配好。
- 应用商店问题经常是多个指标互相拉扯，不是一个计划能全吃满。

### 先看什么

1. 平台是苹果、华为、vivo、oppo 还是小米。
2. 当前主要靠关键词、普通分发、人群包，还是榜单 / ASO。
3. 首日 ROI、付费率、留存、注册成本、自然量分别怎样。
4. 素材展示位、图标、评论、类目有没有协同。

### 当前最不建议做的事

- 把应用商店只当成下载竞价看。
- 只看首日 ROI 就判断计划长期能不能跑。
- 一个计划想同时吃满量级、留存、付费率和 ROI。

### 如果是我来处理

1. 先拆清各计划类型的角色。
2. 再看前后端数据关联和自然量联动。
3. 最后决定是做结构对冲、补 ASO，还是重做人群与素材。

## 7. 团队总定位不到真实问题

### 更像是什么

- 大概率不是单点能力差，而是问题拆解方式错了。
- 更常见的是管理层不看细节、数据体系不完整、考核把动作逼变形。

### 先看什么

1. 有没有清楚分前端、后端、口径、组织四层问题。
2. 管理层有没有定期俯身看数。
3. 数据链路、埋点、回传、归因有没有缺口。
4. 一线有没有试错空间，还是天天被结果倒逼。

### 当前最不建议做的事

- 一出问题就怪素材、怪账户、怪优化师。
- 上面只要结果，不让下面解释过程。

### 如果是我来处理

1. 先把问题按层拆开。
2. 再明确每一层谁负责、缺什么数据、缺什么动作。
3. 最后再谈是继续投、停计划、补系统，还是重做协同机制。

## 8. 招投放负责人 / 招投手时总怕招错人

### 更像是什么

- 真问题通常不是简历少，而是招聘标准太虚、太迷信年限，或者只看历史消耗。
- 很多人吃过红利，但未必真的有归因和带队能力。

### 先看什么

1. 是招执行投手，还是招负责人。
2. 候选人有没有完整讲清失败案例、复盘逻辑和调整过程。
3. 他能不能把素材、流量、转化、回收串成一个闭环。
4. 如果是负责人，他有没有搭体系、带团队和跨部门协同能力。

### 当前最不建议做的事

- 只看做了几年、花过多少钱。
- 只听“我以前跑得很好”，不追问怎么判断、怎么修正。

### 如果是我来处理

1. 先把岗位职责拆成执行能力和组织能力两层。
2. 再让候选人讲失败案例，看逻辑和归因深度。
3. 如果阶段上招不到合适负责人，就先考虑借外脑补诊断和带人。

## 9. 述职 / 复盘怎么写

### 更像是什么

- 不是单纯汇报花了多少钱。
- 真正有价值的是“你比别人看懂了什么、你比过去进步了什么、你接下来打算怎么干”。

### 先写什么

1. 基础数据：消耗、效果、利润，按产品 / 渠道拆开。
2. 对比同行：行业水平、竞品判断、自己的优势和不足。
3. 对比过去：上半年 / 去年到现在的进步。
4. 后续计划：投放技术、产品节奏、行业趋势、具体动作。

### 当前最不建议做的事

- 全篇只有消耗和 revenue。
- 没有趋势判断，没有下一阶段计划。

### 如果是我来处理

1. 先讲结果。
2. 再讲对比和进步。
3. 最后落到趋势和计划，让汇报有“下一步”。

## 10. 海外金融 / 现金贷项目该不该进某个国家

### 更像是什么

- 真问题通常不是“市场够不够大”，而是“牌照、利率、支付、催收和竞争能不能让你活下来”。
- 金融出海不是追热度，很多国家看着热，其实利润和合规都很挤。

### 先看什么

1. 牌照路径、利率上限、外资进入限制。
2. 本地身份认证、支付、征信和三方数据基础设施。
3. 节日、宗教、语言和催收边界。
4. 流量获取成本、头部玩家、生态玩家强度。
5. 资金成本、税费、利润汇回和汇率风险。

### 当前最不建议做的事

- 只看下载榜和活跃榜就决定进场。
- 把印尼、墨西哥、泰国的经验互相硬套。

### 如果是我来处理

1. 先排除牌照和利率逻辑不成立的国家。
2. 再看本地支付、身份和数据基础设施够不够支撑冷启动。
3. 最后才决定做小试点、合作切入，还是直接放弃。

## 11. 海外现金贷冷启动，风控先搭哪几个最小单元

### 更像是什么

- 真问题往往不是模型不够强，而是闭环还没跑通。
- 冷启动阶段最怕大而全，最值钱的是最小单元验证。

### 先看什么

1. 风控画布：客群是谁、要防谁、坏账容忍度到哪。
2. 本地基础数据：身份、三方、设备、支付、GPS、通讯录能拿到哪些。
3. 首贷规则：先挡哪些高风险样本。
4. 人工复核和样本回流有没有安排。
5. 通过率、坏账率、放款量的初始阈值怎么设。

### 当前最不建议做的事

- 一上来就堆大模型和一堆规则。
- 把别国现成策略整包搬过来。

### 如果是我来处理

1. 先用一张风控画布把产品、客群和坏账容忍度讲透。
2. 再搭规则优先的首贷策略和反馈回路。
3. 小样本跑通后，再决定要不要上更复杂的模型和更多数据。

## 12. 风控模型上线几个月就失灵

### 更像是什么

- 更像人群和数据分布漂移，不是模型突然失智。
- 也可能是三方数据衰减、欺诈升级、节日季节性或监控缺位。

### 先看什么

1. PSI、KS、AUC、通过率、拒绝率、坏账率分别怎么变。
2. 哪个国家、渠道、客群先漂了。
3. 三方数据覆盖率、字段稳定性和输入质量有没有变。
4. 最近有没有节日、政策、投放结构和欺诈手法变化。
5. 线上阈值和规则是否被临时改乱。

### 当前最不建议做的事

- 不做归因就直接全量重训。
- 把规则、模型、定价一起乱改。

### 如果是我来处理

1. 先定位是样本漂移、特征失效还是输入脏了。
2. 再用挑战者模型和小样本灰度验证修复方向。
3. 确认有效后再逐步放大，不要一次性全量切换。
